// about

Reliable systems start with a test suite you can trust

Over the last six years at KONE, I've owned test strategy end to end for elevator IoT devices connecting to AWS — deciding what to test, why, and how, based on release risk and system behaviour. I write test code the way I'd write application code: modular, reviewed, version-controlled, and built to outlast the release it shipped for.

Before that, I automated end-to-end regression for set-top boxes at Tata Elxsi, working on a Robot Framework and Selenium suite that ran nightly through Jenkins with no manual babysitting. Across both roles, the constant has been the same: find the fragile manual process, understand why it's fragile, and automate it properly.
