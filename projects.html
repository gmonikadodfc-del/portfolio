IoT device-to-cloud test automation framework

KONE Corporation · 2020–2026
production

The core framework behind release testing for elevator devices connecting to AWS.

Problem

Elevator IoT devices needed reliable component, integration, and system-level testing before every release, but environment setup — device onboarding, PKI provisioning, SSH configuration — was manual and fragile.

Approach

Built a Python and Robot Framework suite treated like production code: modular, reviewed, version-controlled. Automated onboarding, certificate provisioning, and configuration, and containerised environments with Docker for reproducible local runs.

Result

Fewer environment failures ahead of releases, and a test suite the team could extend with confidence rather than route around.

Python Robot Framework AWS PKI Docker

GitLab–JIRA Xray integration

KONE Corporation
key achievement

Closing the gap between automated test runs and the people who need to see the results.

Problem

Automated test results lived in the CI pipeline, disconnected from the JIRA tickets that developers and product owners actually tracked — traceability required manual cross-checking.

Approach

Integrated GitLab automation pipelines directly with JIRA Xray, so test execution results flowed back into tickets automatically.

Result

End-to-end traceability and real-time visibility of automated test results for developers and product owners, without manual reporting.

GitLab CI/CD JIRA Xray Traceability

Docker–Kafka–CloudWatch monitoring pipeline

KONE Corporation
key achievement

Making failures in a distributed streaming system visible instead of guessed-at.

Problem

Diagnosing issues in the Kafka-based streaming layer meant digging through distributed logs with no central view of event flow or failures.

Approach

Built a Docker-based Kafka monitoring solution that published event logs to AWS CloudWatch, giving the team a single place to look.

Result

Set-top box end-to-end regression suite

Tata Elxsi — Client: INVIDI Technologies, Canada · 2017–2019
production

Nightly regression that ran itself, so the team found regressions before customers did.

Problem

Set-top box functionality needed consistent end-to-end and regression coverage across releases, with test specs and risk areas to define from scratch.

Approach

Wrote automated test scripts in Robot Framework and Selenium, wrote test plans in qTest, and configured Jenkins CI to run full nightly regression batches with SQL-based database validation.

Result

Fast, reliable regression feedback every morning without manual intervention, and clear traceability between requirements and test coverage.

Robot Framework Selenium Jenkins qTest SQL
